Super Trend Trading System Zerodha

Long-only automated trading engine for NSE Equity & F&O segments via Zerodha Kite API

2 Indicators

Supertrend + ATR — each individually toggleable

5 Exit Rules

Target, Fixed SL, Trailing SL, End of Session, Supertrend Red

2 Segments

Equity (MIS) in Week 1, Futures & Options in Week 2

Broker: Zerodha Kite Connect

All orders placed via Kite Connect API v3. Requires valid API key + access token. Supports regular, bracket, and cover orders. Market data via WebSocket (Kite Ticker).

Entry Indicators

Each indicator can be independently enabled or disabled. The system adapts its confluence logic based on which indicators are active.

Supertrend Long-only signal — generates BUY when Supertrend turns green (price crosses above ST line). Exit when ST turns red. ENABLED
ATR Volatility filter & dynamic SL sizing — filters low-volatility periods, sets stop loss as ATR multiple ENABLED

Confluence Rules (Long Only)

Active Indicators BUY Entry (Supertrend turns Green) EXIT (Supertrend turns Red)
ST only Enter LONG when Supertrend flips green (price closes above ST line) Exit position when Supertrend flips red (price closes below ST line)
ST + ATR ST flips green AND ATR > threshold (sufficient volatility) Exit when Supertrend flips red (ATR not required for exit)
ATR only ATR alone cannot generate entry signals — it only acts as a filter/gate. Supertrend must be enabled for signal generation.
Long Only — No Short Selling

This is a long-only strategy. The system only takes BUY positions. When Supertrend turns red, the position is exited — no short/sell positions are opened. ATR acts as a volatility filter for entries only.

Exit Conditions

Comprehensive exit management ensures capital protection. Multiple exit conditions are evaluated every tick; the first triggered condition executes the exit.

Exit Priority Order

Daily Auto-Login at 8:30 AM IST

The system automatically logs into Zerodha every day at 8:30 AM IST (before market opens at 9:15 AM). Uses the TOTP key to generate the 2FA OTP programmatically — no manual intervention needed. The login flow obtains a fresh access_token and initializes the Kite session for the day.

Reference Take reference from the Trade Distributor project for the auto-login implementation (TOTP-based Kite login flow).
